# 移动云盘影视 转 CAS 瘦身工具（v3.0）

把移动云盘里**已有的**原视频，零流量变成几 B 的 `.cas` 文件，然后删掉原视频腾出空间；
播放时从 `.cas` 临时恢复出原视频。全部在浏览器里用鼠标操作，**不需要命令行、不需要改代码、不需要联网装任何东西**。

---

## ⚠️ 先看这 3 条重要说明（避免翻车）

1. **本程序主流程只用 Python 标准库，零额外依赖。** 只要设备里有 Python3 就能跑；用 Docker 则连 Python 都不用装（镜像已内置全部所需）。本地直接跑的话 `python3 app.py` 即可，无需 `pip install` 任何东西。
2. **接口是逆向推测、未真机实测。** 移动139 的接口路径/字段基于公开逆向分析推测，可能与官方实际实现有出入。
   如果运行报错，网页右下角"原始接口返回"会显示真实返回，可据此对照 `yidong.py` 顶部的接口路径（`list_path` 等）自行核对；如需协助请在仓库提 Issue 反馈。
3. **CAS 不是备份。** 它依赖网盘底层的"去重池"——热门影视云端有副本，删索引留数据，恢复=秒传建引用；
   **冷门/独一份的文件去重池可能没有，删了就真丢了。** 建议先拿一部热门电影试，确认能恢复再批量删。
4. **天翼(189)暂未完成。** 已知天翼(189)不支持秒传上传，CAS 恢复依赖秒传，所以天翼这条路目前不保证可行
   （详见下面"关于天翼"一节）。本版只支持移动云盘(139)。

## 一、准备：确认有 Python3

> 如果你用 **Docker 部署（见第七节）**，这步和下面的 Python 启动都不需要，跳过即可。

打开终端（Linux 用系统终端，Windows 用 PowerShell 或命令提示符），输入：

```
python3 --version
```

- 显示 `Python 3.x` → 已有，跳过下一步。
- 提示找不到命令 → 安装：
  - Debian / Ubuntu 系：`sudo apt install python3`
  - 其它 Debian 系：`sudo apt update && sudo apt install python3`

> 注意：这里用的是系统自带的 `apt` 源（国内通常可直连），**不是 pip、不是 Docker**，所以不受网络报错影响。

## 二、把程序放到机器上

把 `casgen` 这个文件夹整体拷过去：
- **在本机直接跑（最简单）**：下载仓库的源码压缩包，解压到任意目录（比如 `casgen`）。
- **在其它设备上跑**：把 `casgen` 文件夹传到目标机器的某个目录（比如 `/opt/casgen`）。

文件夹里只需这 5 个文件：`app.py`、`yidong.py`、`share139.py`、`index.html`、`README.md`。

## 三、启动（两种方式，任选其一）

### 方式 A：本机直接跑（推荐，最省事）
1. 终端进入文件夹：`cd casgen`（路径按你实际解压位置改）
2. 启动：`python3 app.py`
3. 浏览器打开：`http://localhost:5000`

### 方式 B：在另一台设备上跑，用本机浏览器访问
1. 在目标设备终端进入文件夹：`cd /opt/casgen`
2. 启动：`python3 app.py`
3. 在另一台设备的浏览器打开：`http://目标设备IP:5000`
   （目标设备 IP 在路由器或系统网络设置里能查到，例如 `http://192.168.1.50:5000`）

> 启动后终端会显示"CAS 转换服务已启动"。要停止就按 `Ctrl+C`。

## 四、网页里怎么用（全鼠标）

1. **登录**：选"移动云盘(139)"，把浏览器的 `Authorization` 头粘进文本框，点"登录并加载根目录"。
   - 怎么拿 Authorization：用电脑浏览器登录 `yun.139.com` → 按 F12 打开"开发者工具" → 切到 Network(网络) 标签 →
     随便点一个网盘请求 → 在请求头(Request Headers)里找 `Authorization: Bearer xxxx` → 把整串 `Bearer xxxx` 复制下来粘进去。
2. **选文件夹**：在目录树里一层层点进你想处理的影视文件夹，点"✔ 选定此文件夹用于转换"。
3. **先分析**：点"① 先分析"，会告诉你这个文件夹里有多少个能转、多少个跳过（云端没 SHA256 的跳过）。
4. **转换**：**先不要勾"删除原视频"**，点"② 开始转换"。它会零流量生成 `.cas` 文件（几 KB）。
5. **验证恢复**：在"第 4 步"里点"恢复"，确认网盘里能重新生成原视频。
6. **确认无误后再批量删**：回到第 3 步，**勾上"删除原视频"**再点一次转换，才会删源腾空间。

## 四·五、分享链接一条龙（2.0 新增）

别人分享给你的移动云盘(139) 链接，可以直接在这个工具里解析、把里面的视频一键转存到你自己的云盘，之后再按上面的「生成 CAS」瘦身，形成一条龙。

1. **粘贴链接**：在「📥 分享链接一条龙」卡片里，把别人的分享链接粘进输入框。**提取码不用单独填**——只要链接文字里带 `提取码:xxxx` / `密码:xxxx` / `?pwd=xxxx` / `&pwd=xxxx`，程序会自动识别一起解析。
2. **解析**：点「① 解析并列出视频」，会列出分享里所有的视频文件（自动过滤掉字幕/图片等非视频）。
3. **转存**：填一个「转存到我云盘的目录」（留空=根目录，可填目录名或路径如 `/电影/分享`），点「② 转存视频到我云盘」，视频就零流量转存进你的云盘了。
   - **两个可选复选框（省一步路）**：
     - ☑ **转存后自动生成 CAS**：勾上后，转存完会直接对该目录生成 `.cas`，不用再手动回到「选文件夹」去点转换，一条龙搞定。
     - ☑ **生成 CAS 后删除原视频**：勾上「自动生成 CAS」后才可勾；勾上后生成 `.cas` 的同时删掉原视频腾空间（冷门文件慎用，先确认能恢复）。
4. （若没勾自动生成 CAS）之后回到「选文件夹」选你刚转存的目录，按上面的步骤生成 `.cas` 瘦身即可。

> 说明：分享解析走的是移动云盘 139 的明文 V6 接口（无需额外签名）；`share139.py` 里另带一个 `OutLinkCrypto` 加密外链备用类（基于 pycryptodome 的 AES-128-CBC），供后续需要时解析官方 App 的加密外链。该备用类采用**惰性导入**——`pycryptodome` 是可选依赖，未安装时仅此类不可用，**主流程（解析/转存/生成 CAS）完全不受影响**。

## 四·六、分享链接定时监控（3.0 新增）

针对**还在更新中的电视剧/剧集分享链接**：添加监控后，工具会按设定间隔自动去检查这个链接，发现里面**多了新文件**（如更新的几集），就自动把新增部分转存到你指定的云盘目录，并按需自动生成 CAS（可选同时删除原视频）。追剧党不用再手动反复转存。

1. **添加监控**：在「📡 分享链接定时监控」卡片里粘贴分享链接（提取码同样自动识别），填「转存到我云盘的目录」（留空=根目录），设一个扫描间隔（**最短 60 分钟**，默认 60），勾选是否自动生成 CAS / 删原视频，点「＋ 添加监控」。添加时会立即拉一次链接建立基线（记录当前已有文件）。
2. **自动差异检测**：后台线程按间隔轮询，用 contentID 集合做差集，只转存「链接里新增、你云盘里还没有」的文件，不会重复转存已有剧集。
3. **链接失效 / 登录失效处理**：
   - 分享链接本身失效（过期/被取消）→ 该项标记「链接失效 ✖」并停止扫描，不影响其它监控。
   - 移动云盘登录态过期 → 全局弹红色横幅提示「登录已失效，请重新登录」，所有监控暂停；你重新登录后后台自动恢复监控。
4. **立即扫描 / 删除**：列表里每个监控项可手动「立即扫描」或「删除」。页面每 60 秒自动刷新监控状态（状态 / 上次扫描时间 / 最近一次结果）。

> 提示：监控转存后云盘需要一点时间索引，程序已内置 30 秒等待 + 失败重试（最多 3 次）来保证 CAS 生成成功。扫描间隔下限 60 分钟是基于风控考虑，请勿调更低。

## 五、关于天翼(189)

移动(139)的 CAS 恢复依赖"按哈希秒传"，已知**天翼(189)不支持秒传上传**，
所以天翼的"零流量生成 CAS + 恢复"目前找不到可行路径。因此本版只做了 139。
如果未来需要支持天翼，需换思路（比如浏览器复制 cookie 登录 + 依赖天翼自己的去重），
但成功率未知。新的实现思路欢迎在仓库 Issue 中讨论。

## 六、出错怎么处理

网页右下角"🛠 原始接口返回"里会显示程序收到的真实返回。可据此对照 `yidong.py` 顶部的接口路径（`list_path` 等）自行核对；
如需协助，把那段返回内容在仓库提 Issue 反馈，维护者会据此调整接口路径，使用者无需改动代码。

## 七、用 Docker 部署（飞牛 FnOS / 群晖 / 任意支持 Docker 的设备 —— 推荐）

镜像由本仓库的 **GitHub Actions 自动构建并公开推到 Docker Hub**，所以你在支持 Docker 的设备上就能直接拉取运行，不用装 Python、不用管依赖。

### 路线 A：飞牛 FnOS Docker「镜像仓库」直接搜到安装（推荐路线）
1. 飞牛应用中心装好「Docker」应用（系统自带）。
2. 打开 Docker → **镜像仓库** → 搜索 `tianjian518/casgen`
   （镜像由本仓库维护者构建并公开托管，直接搜这个名字即可）。
3. 搜到后点「拉取」，等进度条走完。
4. 切到「容器」→「创建容器」→ 镜像选刚拉取的 `tianjian518/casgen` →
   **端口映射**填：容器端口 `5000` → 主机端口 `5000` → 确定。
5. 浏览器打开 `http://飞牛IP:5000` 即可用。

> 镜像由仓库维护者在每次发布时自动构建，通常 1~2 分钟即就绪；面向使用者无需关心构建过程，直接拉取使用即可。

### 路线 B：用 docker compose（电脑/设备命令行均可）
把本仓库的 `docker-compose.yml` 放到任意目录，执行：
```bash
docker compose up -d        # 直接拉 Docker Hub 镜像运行
# 或本地从源码构建： docker compose up -d --build
```
浏览器开 `http://IP:5000`。

## 版本说明

- **v3.0.0（当前）**：新增「分享链接定时监控」——对还在更新的剧集分享链接按间隔自动检查、自动转存新增文件并生成 CAS（差异检测基于 contentID 集合，不重复转存；链接失效优雅标记停止、登录失效弹横幅暂停并提示重登后自动恢复）；**补齐登录态失效检测与重登提示**（刷新页面不丢登录，登录过期前端弹红色横幅）；目录结构新增 `monitor.py` / `monitor_store.py`（纯标准库，零新依赖）。
- **v2.0.0**：新增「分享链接一条龙」（解析别人分享的 139 链接、自动识别提取码、一键转存视频到自己的云盘），并支持**转存后自动生成 CAS**、**生成后删除原视频**两个可选复选框，一条龙省一步；新增视频后缀白名单（只转换 MP4/MKV 等视频，跳过图片/文本）；修复重复生成 CAS（`auto_rename`→`overwrite`）与 `.cas` 后缀叠加问题；**去掉强制加密依赖**（注：登录态持久化当时规划但未落地，实际在 3.0 补齐）——`pycryptodome` 改为可选（仅 `OutLinkCrypto` 备用类按需惰性导入），主流程零额外依赖、Docker 镜像更轻、三架构构建更稳。
- **v1.0.0**：首个稳定版，支持移动云盘(139) 影视零流量转 CAS、Docker 多架构镜像（amd64/arm64/armv7）。

> **每个版本都有独立镜像标签，可任选其一拉取**：`tianjian518/casgen:latest`（始终指向最新 main），以及每个发版的 `:x.y.z` 与简写 `:x.y`——例如 `:3.0` / `:3.0.0`、`/:2.0` / `:2.0.0`、`/:1.0` / `:1.0.0`。推送 `v*` git tag 即由 GitHub Actions 自动构建对应镜像标签。想固定某个版本就拉对应标签，例如 `docker pull tianjian518/casgen:2.0`。
